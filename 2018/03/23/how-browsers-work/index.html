浏览器结构

第一个 web 浏览器是单片程序, 但浏览器开发人员很快就采用了模块化的方法, 将浏览器拆分为几个大组件:

  • 宿主应用程序嵌入其他浏览器组件, 充当它们、用户和主机操作系统之间的接口。宿主应用程序包括围绕网页内容的 GUI, 如选项卡栏、地址栏和书签管理器。
  • 呈现引擎用于呈现网页的大部分工作。当用户访问页面时, 宿主应用程序为呈现引擎提供 URL 和内容窗口矩形坐标。然后, 呈现引擎检索 HTML 文档和相关资源 (如样式表和图像), 并在给定矩形中绘制该页的图形表示形式。引擎还处理链接、窗体和相关详细信息。
  • 现代浏览器通常使用专用引擎执行 JavaScript, 例如 Google Chrome 的 V8 或火狐的 SpiderMonkey。它与呈现引擎相互独立, 但它们共享每个标签页的 DOM 数据结构。

这种模块化方法的优点是允许呈现引擎嵌入到其他程序中, 例如, 在电子邮件客户端中呈现HTML 电子邮件或 EPUB 电子书阅读器。(JavaScript 引擎也可以被嵌入到其他程序中, 例如 V8 中的 node. js 和 GNOME 3 中的 SpiderMonkey .)

浏览器主要组件：

  1. 用户界面： 包括地址栏，前进后退按钮，书签菜单等

  2. 浏览器引擎：在用户界面和呈现引擎之间传送指令

  3. 呈现引擎：负责显示请求内容

  4. 网络：用于网络调用

  5. 用户界面后端：用于绘制基本的窗口小部件，比如组合框和窗口。其公开了与平台无关的通用接口，而在底层使用操作系统的用户界面方法

  6. javascript解释器：解析执行js代码

  7. 数据存储：浏览器需要在硬盘上保存的各种数据（cookie等），这是持久层。HTML5定义了‘网络数据库’，这是一个完整的浏览器数据库

浏览器引擎（呈现引擎）

​ 浏览器引擎，也叫呈现引擎、浏览器内核。呈现引擎由各浏览器厂商按W3C标准自己研发。有以下几种：Webkit，Geocko，Trident，Blink。

  • Trident：IE内核，也被用于Microsoft Outlook和其他一些Windows应用程序。然而现在Edge浏览器用的是新的浏览器内核，叫做EdgeHTML
  • Geocko：Mozilla使用的呈现引擎，Firefox的内核，也用在Thunderbird e-mail 客户端中。Geocko有一个叫 Goanna)的分支，现在被用在 Pale Moon) 浏览器
  • Webkit： Safari内核，Chrome以前的内核
  • Blink： Blink 其实是 Webkit 的一个分支，添加了一些优化的新特性，例如跨进程的 iframe，将 DOM 移入 JavaScript 中来提高 JavaScript 对 DOM 的访问速度等，目前较多的移动端应用内嵌的浏览器内核也渐渐开始采用 Blink。由 Google 和 Opera Software 开发，在Chrome（28及往后版本）、Opera（15及往后版本）和Yandex浏览器中使用。（Opera 以前用的是自己的 Presto 引擎）

浏览器解释并显示 HTML 文件的方式是在 HTML 和 CSS 规范中指定的。这些规范由网络标准化组织W3C进行维护。

渲染流程

​ 简单来说，浏览器渲染流程主要如下：

  1. 获取文档内容： 呈现引擎从网络层获取请求文档的内容（内容的大小一般限制在 8000 个块以内）
  2. 生成DOM Tree：呈现引擎开始解析HTML文档，把各个标记转成DOM Tree 的节点
  3. 创建Render Tree： 同时解析外部css文件和样式元素中的样式数据，这些数据将用于创建Render Tree。Render Tree包含多个带有视觉属性的矩形，矩形的排列顺序就是在屏幕上显示的顺序
  4. 布局 ：Render Tree 构建好后，就开始为每个节点分配在屏幕中出现的坐标
  5. 绘制：呈现引擎遍历Render Tree，由用户界面后端层将每个节点绘制出来

​ 为达到更好的用户体验，呈现引擎会力求尽快将内容显示在屏幕上。它不必等到整个 HTML 文档解析完毕之后，就会开始构建呈现树和设置布局。在不断接收和处理来自网络的其余内容的同时，呈现引擎会将部分内容解析并显示出来。
